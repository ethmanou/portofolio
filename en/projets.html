WAF (ModSecurity)

Application Security / 02/2025

Objectives

Protect exposed web applications from common attacks (injections, XSS, CSRF) while minimizing false positives and performance impact.

Detailed Actions

  • Architecture: Deployed ModSecurity as a reverse-proxy (Docker) in front of multiple application backends; integrated with the CI pipeline for automated testing.
  • Rules: Enabled OWASP CRS as a baseline, then created custom rules for business-specific cases (upload endpoint exceptions, fine-tuned REST API rules).
  • Tuning: Collected and analyzed logs for 2 weeks in a staging environment, created whitelists/exceptions, and progressively switched to blocking mode.
  • Monitoring: Centralized logs via Filebeat → ELK / Grafana for alert correlation and metrics (requests/blocks, false positive rate).
  • Testing & Validation: Conducted black-box tests on staging (validation scenarios, non-exhaustive) plus load tests to measure latency impact.

Results & Metrics

  • Confirmed blocking of injections and XSS sequences in the test environment.
  • Gradual reduction of false positives after tuning and implementing targeted exceptions.
  • Set up an operational dashboard displaying processed requests, block rate, and average latency.

Skills & Tools Gained

ModSecurity (reverse-proxy & integrated mode), OWASP CRS tuning, log parsing and correlation (ELK/Grafana), application testing and validation, DevOps culture (containerized deployment).

Active Directory Hardening

IAM & Systems / 11/2024

Objectives

Harden the AD infrastructure to reduce the attack surface related to identity management and secure administrative access.

Detailed Actions

  • OU Design: Defined a clear organizational structure (OUs per service/environment) to facilitate targeted GPOs and minimal privilege delegation.
  • GPO & Hardening: Created standardized GPOs: password policies (length & complexity), account lockout, expiration, deactivation of inactive accounts; configured detailed logging.
  • Privilege Management: Separated administrative accounts (Break-glass), applied least privilege principle, and controlled delegation via groups and roles.
  • MFA & Access: Integrated MFA for sensitive access; reviewed services relying on service accounts.
  • Monitoring: Centralized logs via Windows Event Forwarding → SIEM (event correlation, alerts on suspicious behavior).
  • Processes: Implemented patching procedures and an incident response playbook for AD compromise scenarios.

Results & Metrics

  • Improved access governance: documented permissions and regular audits.
  • Faster anomaly detection through SIEM alerts on critical AD events.
  • Tested recovery procedures (AD object restore, critical account reset).

Skills & Tools Gained

Active Directory (AD DS), GPO design & hardening, IAM, Windows Server administration, audit & SIEM, patch management procedures, and incident response playbooks.

Network Migration & Hardening

Networks & Security / 09/2024

Objectives

Redesigned the network architecture to improve availability, performance, and security while minimizing downtime, in preparation for the CSNA certification, which I successfully obtained.

Detailed Actions

  • Design: Mapped network flows, implemented logical segmentation via VLANs (e.g., Management VLAN, Production VLAN, DMZ VLAN), and defined trust zones.
  • Perimeter Security: Deployed a Stormshield firewall with deny-by-default policies, and explicit rules for exposed services (HTTPS, restricted SSH management).
  • Remote Access: Configured site-to-site IPsec VPNs and user VPNs (encrypted tunnels, strong authentication, controlled split-tunnel).
  • Routing & High Availability: Set up OSPF for dynamic routing between sites and redundancy of critical links; implemented granular ACLs to filter inter-VLAN traffic.
  • Testing & Migration: Executed phased migration (staging → rollback plan → production), performed load testing, and monitored the network during cut-over.

Results & Metrics

  • Segmented network limiting lateral propagation of incidents.
  • Encrypted and reliable remote access via VPN; granular firewall rules enforced.
  • Documented and tested migration process; monitoring in place for latency and availability.

Skills & Tools Gained

VLAN, OSPF, ACL design, IPsec VPN, Stormshield administration, resilient network architecture design, migration procedures and validation testing, as well as preparation for and successful attainment of the CSNA certification.
